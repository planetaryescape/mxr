## The problem

Every time you open Gmail, you leave your terminal. You type into a laggy textarea with none of the muscle memory you've spent years building. No vim motions. No macros. No composability.

mutt solved this in 1995. It still works, but it feels like it. aerc is a step forward but doesn't reimagine the experience. himalaya is a CLI tool, not a full client.

**mxr sits in the gap.**

</div>

<div class="landing-section features-section">

## What makes mxr different

<CardGrid stagger>
  <Card title="Compose in your editor" icon="pencil">
    `$EDITOR` opens with markdown and YAML frontmatter. Write email the way you write everything else. Full vim motions, your plugins, your config.
  </Card>
  <Card title="Search, don't browse" icon="magnifier">
    Tantivy gives you BM25-ranked, sub-second search across your entire mailbox. Search is navigation, not an afterthought bolted onto a folder tree.
  </Card>
  <Card title="Distraction-free reading" icon="open-book">
    mxr shows you what the email says, not what it's trying to sell you. No tracking pixels. No banners. Just the words. One key to unsubscribe.
  </Card>
  <Card title="Daemon-backed" icon="setting">
    Sync, indexing, rules, and snooze run in the background. Close the TUI — nothing stops. CLI and scripts talk to the same engine.
  </Card>
  <Card title="Local-first" icon="laptop">
    SQLite is the canonical store. Your email lives on your machine. Works offline. Cloud services are transports, not dependencies.
  </Card>
  <Card title="Scriptable" icon="seti:shell">
    Every TUI action has a CLI equivalent. Pipe data to shell commands. Unix composition over plugin frameworks.
  </Card>
</CardGrid>

</div>

## Install

```bash
# Homebrew
brew tap planetaryescape/mxr && brew install mxr

# Cargo
cargo install mxr-daemon

# Pre-built binaries
curl -fsSL https://github.com/planetaryescape/mxr/releases/latest
```
