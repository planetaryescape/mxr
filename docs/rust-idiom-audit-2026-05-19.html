Main blockers before I would call the workspace idiomatically clean

Short version: The model boundaries, typed IDs, provider adapters, SQLite/search split, and test investment are above average. The non-idiomatic parts are visible in automation: cargo fmt --all --check fails, workspace Clippy is noisy, and mxr-semantic has a feature-gated test compile failure. That makes the repo feel less clean than the architecture deserves.

Top Findings

P1

The local semantic test target does not compile under the feature that is supposed to exercise it.

Default scripts/cargo-test -p mxr-semantic --tests ran zero tests; --features local failed with a stale Envelope fixture missing keywords. The normal app build with semantic-local passes, so this is test drift rather than product compile failure, but it weakens confidence in the most optional and dependency-heavy layer.

Evidence: crates/semantic/src/lib.rs:1268, compiler error E0063 missing field keywords.
P1

Formatting is not clean.

cargo fmt --all --check fails across daemon, store, provider, search, sync, TUI, and mail parsing files. In Rust projects this is normally a baseline invariant, not an aspirational cleanup task.

Evidence: formatter drift in crates/daemon/src/handler/mod.rs, crates/store/src/lib.rs, crates/tui/src/lib.rs, crates/provider-imap/src/lib.rs, and others.
P2

Workspace Clippy is not close to clean.

The strict run with -D warnings fails early. The non-strict run surfaced hundreds of warnings plus a feature-gated compile error. The notable production-facing lints are large enum variants, public panics, wrong-self convention, methods named from_str without implementing FromStr, too-many-arguments APIs, and detached futures.

Evidence: crates/protocol/src/types.rs:329, crates/core/src/i18n.rs:92, crates/daemon/src/loops.rs:37, crates/store/src/search.rs:81.
P2

Some public/core functions encode impossible states with panic! instead of the type system.

The iCalendar response helpers accept a broad participation enum and panic for non-sendable variants. Idiomatic Rust should narrow the input type, return Option/Result, or model sendable values separately.

Evidence: crates/core/src/i18n.rs:84-121.
P2

The daemon, TUI, protocol, and web surfaces have several "god object" pressure points.

The largest files are not just big; they sit on critical integration paths. The long files also correlate with Clippy warnings like large_enum_variant and too_many_arguments, which are usually signs that boundaries want another pass.

Evidence: crates/tui/src/lib.rs 11,130 LOC, crates/daemon/src/handler/mod.rs 10,306 LOC, crates/web/src/lib.rs 6,096 LOC, crates/protocol/src/types.rs 3,374 LOC.
P2

One background task is explicitly detached without a retained handle.

let _ = tokio::spawn(...) for the idle watcher loses shutdown/error visibility. That is not automatically a bug, but it conflicts with Tokio's graceful shutdown guidance and the daemon's operational posture.

Evidence: crates/daemon/src/loops.rs:37.
P2

Architecture docs have drifted behind newer crates.

The documented dependency rules are mostly honored by the original core/provider/store/search/sync split, but newer crates like relationship, safety, llm, and provider-outlook are not fully captured. This makes future review harder than it needs to be.

Evidence: relationship -> store, llm; safety -> relationship; newer crates absent from the project architecture section.
P3

Dependency hygiene is active, but not quiet.

cargo deny check passes, which is good. It also reports a yanked transitive crate, many duplicate versions, and stale advisory ignores. That is acceptable for now, but it should not be allowed to become background noise.

Evidence: yanked core2 0.4.0 via the image/semantic stack; stale ignores for RUSTSEC IDs no longer encountered.

Reading guide

The highest scores cluster around search, sync, provider tests, and persistence. The lowest scores cluster around feature-gated semantic tests, maintainability of giant integration files, and hygiene gates.

These scores are intentionally conservative. A crate can pass tests and still score low if its current shape makes future changes risky.

Per-Crate Notes

What To Fix First

1. Restore hygiene gates

Fix formatter drift, make semantic local tests compile, and decide which Clippy lint set is merge-blocking. This is the fastest way to make the repo feel like idiomatic Rust again.

2. Remove production panics

Start with core::i18n, then production unwrap/expect in daemon commands and store helpers. Tests can keep explicit expect when it improves failure messages.

3. Split the integration bulk

Break the biggest handler/TUI/web/protocol surfaces along request or domain boundaries. The goal is not aesthetic smallness; it is cheaper review, smaller blast radius, and fewer giant enum variants.

4. Box large variants

Fix the large_enum_variant warnings in protocol, TUI async results, and CLI action enums where payload size is materially inflated by rare variants.

5. Track spawned work

Give daemon background tasks explicit lifecycle ownership. Detached tasks are fine only when the lifecycle and error policy are deliberate and visible.

6. Refresh architecture docs

Add the newer relationship, safety, llm, and provider-outlook crates to the dependency-boundary model, including what each is allowed to know.
